# Sprint 1 Notes — Domain Taxonomy Freeze

**Status:** Complete

**Taxonomy version:** 1.0.0 (frozen)  
**Date:** 2026-07-18  

## Goal

Lock the agriculture domain taxonomy used by all later Mini workers and align platform query understanding with the same aliases.

## Delivered

1. **Frozen taxonomy v1.0.0** (`mini/taxonomy/domains.py`)
   - 14 categories (soil, weather, crop, disease, pest, fertilizer, irrigation, scheme, market, finance, machinery, seed, advisory, general)
   - 22 crops with EN / MR / HI + scientific names
   - 10 crop stages
   - Soil types, irrigation types, fertilizer types, machinery, scheme catalog

2. **Aliases EN/MR/HI** (`mini/taxonomy/aliases.py`)
   - `CROP_ALIASES` + `CATEGORY_ALIASES`
   - `resolve_crop_name` / `resolve_crops_in_text`

3. **Region hierarchy** (`mini/taxonomy/regions.py`)
   - Maharashtra zones + districts, MP/KA/GJ stubs
   - District alias resolve (Pune, नागपूर, Aurangabad → Sambhajinagar, …)

4. **Unit vocabulary** (`mini/taxonomy/units.py`)
   - mass, area, temp, rainfall, currency, price, EC, pH, water volume
   - convert_mass / convert_area helpers

5. **Validator** (`mini/taxonomy/validator.py`)
   - Integrity checks
   - Platform KB coverage: every crop/disease/fert/graph crop maps to taxonomy

6. **TaxonomyService** + workers
   - `W-TAXONOMY` (ready)
   - `W-NORMALIZE` (partial — taxonomy-backed)
   - Pipelines: `taxonomy`, `sprint1`

7. **Platform integration**
   - `query_understanding` imports taxonomy aliases
   - API: `GET /api/taxonomy`, `/api/taxonomy/validate`, `/api/taxonomy/resolve`
   - Streamlit: sidebar taxonomy browser + **Domain Taxonomy** tab

8. **CLI**
   ```bash
   python -m mini.orchestrator taxonomy-summary
   python -m mini.orchestrator taxonomy-validate
   python -m mini.orchestrator run sprint1 --execute
   ```
