# Sprint 8 Notes — Knowledge Graph Builder (W-KGBUILD)

**Status:** Complete  
**Sprint:** S8  
**Feature phase:** FP-4  
**Date:** 2026-07-18  

## Goal

Automated agriculture knowledge graph construction from standard entities, taxonomy, processed lake facts, and the platform seed graph.

## Delivered

1. **`mini/lake/kg_build.py`**
   - Merge seed `data/knowledge_graph.json` + taxonomy + processed lake + standard records
   - Relation templates: `AFFECTED_BY`, `TREATED_BY`, `GROWS_IN`, `REQUIRES_FERTILIZER`,
     `COVERED_BY`, `BENEFITS_FROM`, `STRESSED_BY`, `TRIGGERED_BY`, `HAS_STAGE`,
     `GROWN_IN`, `NEEDS_NUTRIENT`, `USES_IRRIGATION`
   - NetworkX GraphML export
   - Graph triple training text (`graph_triples.jsonl`)
   - Neo4j loader stub (Cypher hints)
2. **`W-KGBUILD`** ready worker + CLI `kgbuild` + API `/api/lake/kg`
3. Pipeline `sprint8` / `kgbuild`
4. Artifacts under **`mini/datasets/`** (not required in `data/` for CI)
   - `mini/datasets/KG_LATEST.json`
   - `mini/datasets/kg/*`
5. Platform seed rewrite is **opt-in** (`--write-seed`) so `data/` stays clean by default
