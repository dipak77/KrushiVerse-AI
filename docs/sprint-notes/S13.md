# Sprint 13 Notes — Evaluation Harness (W-EVAL)

**Status:** Complete  
**Sprint:** S13  
**Feature phase:** E5-eval  
**Date:** 2026-07-19  

## Goal

Block bad Mini checkpoints from “promotion” with an automated scorecard: gold QA, LM loss/PPL, latency/memory, hallucination probes, regional MH checks, and configurable gates. Produce HTML + JSON reports; **failing gates exit non-zero**.

## Delivered

1. **`mini/eval/gold_sets.py`** — disease, fertilizer, schemes, market, regional MH gold items  
2. **`mini/eval/metrics.py`** — token-F1, EM, ROUGE-L, latency p95, RSS memory  
3. **`mini/eval/probes.py`** — safety + fact-conflict hallucination probes  
4. **`mini/eval/gates.py`** — `default` / `strict` promotion profiles  
5. **`mini/eval/harness.py`** — load v0.4 (or v0.3/v0.2), score, write `EVAL_LATEST.json` + `.html`  
6. **`W-EVAL`** ready  
7. CLI `eval --execute --version v0.4 --profile default`  
8. API `GET/POST /api/lake/eval`  
9. Pipeline `sprint13` / `eval`
